# <a name="translate-site-pages"></a>Перевод страниц сайта
Прежде чем начать трансляцию сайта путей обучения, важно понимать несколько ключевых концепций, в которых многоязычный компонент работает с использованием путей обучения. 
- Сведения о сайте — Навигация, логотип и имена сайтов требуют, чтобы сайт просматривался и переводился в языковой профиль пользователя.  
- Веб-часть "пути для обучения" должна просматриваться с использованием языкового профиля пользователя, чтобы он отображался на языке, отличном от английского. Веб-часть и контент, предоставленные корпорацией Майкрософт, уже переведены на вас. Для получения дополнительных сведений о профилях языков ознакомьтесь со статьей [изменение личных языков и региональных параметров](https://support.microsoft.com/en-us/office/change-your-personal-language-and-region-settings-caa1fccc-bcdb-42f3-9e5b-45957647ffd7).
- Способ настройки путей обучения определяет, доступны ли переведенные страницы. Для новых сайтов, подготовленных с помощью службы подготовки SharePoint, будут доступны переведенные страницы на девяти языках. Для обновленных сайтов или сайтов, которые вы создаете, потребуется перевод вручную. Сведения [о параметрах настройки для многоязыковых путей обучения](custom_setupoptions_ml.md).
- Многоязыковая поддержка для путей обучения включена в многоязычных функциях SharePoint Online для сайтов для общения. Чтобы узнать о многоязычных функциях SharePoint Online, ознакомьтесь со статьей [Создание многоязыковых сайтов для общения, страниц и новостей](https://support.office.com/article/2bb7d610-5453-41c6-a0e8-6f40b3ed750c). 

## <a name="working-with-a-newly-provisioned-site"></a>Работа с новым подготовленным сайтом
Если вы задаете новый сайт обучающих путей из службы подготовки SharePoint, переведенные страницы уже доступны вам. По умолчанию сайт предоставляет следующие страницы:

- Home. aspx
- Старт-ВИС-Сикс-Симпле-Степс. aspx
- Get-started-with-Microsoft-365. aspx
- Жет-стартед-ВИС-Микрософт-теамс. aspx
- Жет-стартед-ВИС-шарепоинт. aspx
- Жет-стартед-ВИС-онедрииве. aspx
- АСК-куестионс-Анд-жет-Хелп. aspx
- Обучающие события Calendar. aspx
- Бекоме-а-Чампион. aspx
- Рекоммендед-плайлистс. aspx
- Обучающие материалы — центр успешных административных путей

## <a name="view-translated-pages-from-the-newly-provisioned-site"></a>Просмотр переведенных страниц с нового наполненного сайта
Чтобы ознакомиться с переведенным веб-сайтом путей обучения, рассмотрим несколько преобразованных страниц.

### <a name="view-the-translated-home-page"></a>Просмотр переведенной домашней страницы
На домашней странице пути для обучения выберите язык из раскрывающегося меню язык, как показано в следующем примере. В этом примере отображается итальянский, выбранный в правом верхнем углу, и все элементы страницы переводятся.

![custom_ml_pages_home.png](media/custom_ml_pages_home.png)

### <a name="view-the-translated-microsoft-365-training-page"></a>Просмотр переведенной страницы обучения Microsoft 365
Теперь давайте взглянем на страницу обучения по Microsoft 365. 

1. На **домашней** странице сайта пути для обучения выберите **Microsoft 365 Training**.
2. В правом верхнем углу страницы выберите язык. В этом примере выбран итальянский.

![custom_ml_pages_training.png](media/custom_ml_pages_training.png)

Какие переводы отображаются при выборе языка?
- Страница SharePoint переводится, как показано на рисунке выше. Обратите внимание на то, что текст баннера страницы теперь находится на итальянском языке.

Что не отображается в переводах?
- Имя сайта отображается на английском языке
- Навигация по сайту выполняется на английском языке
- Веб-часть "пути для обучения" на английском языке

## <a name="view-the-fully-translated-site"></a>Просмотр полностью переведенного сайта 
Для просмотра полностью переведенного сайта на определенном языке, в том числе страниц сайта, навигации и веб-части, необходимо настроить персональные язык и региональные параметры пользователя для этого языка. Дополнительные сведения об установке языковых и региональных параметров приведены в статье [изменение личных языков и региональных параметров](https://support.microsoft.com/en-us/office/change-your-personal-language-and-region-settings-caa1fccc-bcdb-42f3-9e5b-45957647ffd7). Для просмотра переведенных страниц рекомендуется использовать отдельную учетную запись или другой пользователь с разными языковыми параметрами.  

## <a name="working-with-an-updated-or-manually-installed-learning-pathways-site"></a>Работа с обновленным или установленным вручную сайтом обучающих путей
Если вы обновили существующий сайт путей обучения или установили веб-часть вручную на существующий сайт, вам потребуется вручную перевести страницы сайта. Веб-часть "пути для обучения" и контент уже переведены и будут отображаться на предпочитаемом пользователем языке. Чтобы преобразовать страницы, ознакомьтесь со статьей "Создание страниц для нужных языков". 

## <a name="create-pages-for-the-languages-you-want"></a>Создание страниц для нужных языков
После того как вы включили свой сайт для многоязычных функций и выбрали языки, которые вы хотите сделать доступными, вы можете создать нужные страницы перевода. 

1. Перейдите на страницу язык по умолчанию, которую нужно сделать доступной на другом языке.
2. На верхней панели нажмите кнопку **перевод**.
3. Выберите пункт **создать** для нужных языков.

> [!IMPORTANT]
> После создания страниц перевода необходимо опубликовать (или повторно опубликовать) страницу языка по умолчанию, чтобы убедиться, что:
>- Страницы перевода отображаются на соответствующем сайте языка.
>- Страницы перевода правильно отображаются в веб-части новостей и выделенных веб-частях контента.
>- Раскрывающийся список язык в верхней части сайта включает все доступные языки.
>- Переводчик получает уведомление о запросе на перевод.

После создания страниц состояние страницы (Черновик сохранен, опубликован и т. д.) отображается в области перевода рядом с каждым языком. Кроме того, указанные трансляторы будут уведомлены в сообщении электронной почты о запросе на перевод.

### <a name="view-the-fully-translated-site-in-a-specific-language"></a>Просмотр полностью переведенного сайта на определенном языке
Для просмотра полностью переведенного сайта на определенном языке, в том числе страниц сайта, навигации и веб-части, необходимо настроить персональные язык и региональные параметры пользователя для этого языка. Дополнительные сведения об установке языковых и региональных параметров приведены в статье [изменение личных языков и региональных параметров](https://support.microsoft.com/en-us/office/change-your-personal-language-and-region-settings-caa1fccc-bcdb-42f3-9e5b-45957647ffd7). Обратите внимание на то, что лучше всего использовать отдельную учетную запись или иметь другой пользователь с другими языковыми параметрами просмотра переведенных страниц.

## <a name="what-does-a-translator-do"></a>Возможности переводчика
 После настройки сайта на английском языке пользователь с испанским языком в качестве предпочтительного личного языка, например, вручную редактирует название, навигацию и содержимое нижнего колонтитула на испанском языке. Пользователь с немецким языком в качестве предпочтительного для немецкого языка. После перевода контента он будет отображаться для всех пользователей этих предпочитаемых языков. Веб-часть выбирает предпочтительный язык пользователя и отображает содержимое, переведенное на этот язык. 

Переводчики вручную преобразуют копии языковой страницы по умолчанию в указанные языки. При создании копий страниц переводчики уведомляются в сообщении электронной почты, если указан транслятор. Сообщение содержит ссылку на страницу языка по умолчанию и созданную страницу перевода. Переводчик будет:
1. Нажмите кнопку **начать перевод** в сообщении электронной почты.
2. В правом верхнем углу страницы нажмите кнопку **изменить** , а затем — перевод контента.
3. Когда все будет готово, выберите **Сохранить как черновик** (если вы не готовы сделать его видимым для читателей) или если страница готова к отображению всем, кто использует этот язык на сайте, выберите **опубликовать** или опубликовать **Новости**.

Дополнительные сведения о процессе перевода можно найти в [статье Создание многоязыковых сайтов для общения, страниц и новостей](https://support.office.com/en-us/article/2bb7d610-5453-41c6-a0e8-6f40b3ed750c). 

## <a name="updating-the-default-language-page"></a>Обновление страницы языка по умолчанию
После обновления страницы языка по умолчанию эту страницу необходимо повторно опубликовать. Затем трансляторы для страниц перевода уведомляются в сообщении электронной почты о том, что обновление выполнено, поэтому обновления можно вносить на отдельные страницы перевода.

## <a name="set-up-a-multilingual-site-name-navigation-and-footer"></a>Настройка имени многоязыкового сайта, навигации и нижнего колонтитула
Для отображения имени сайта, навигации и нижнего колонтитула сайта на разных доступных языках, каждый из них должен быть переведен вручную.

Например, предположим, что вы создали сайт для общения с языком по умолчанию на английском языке, и вы включили сайт для испанского и немецкого языков. Когда вы создаете сайт, вы настраиваете имя и описание сайта на языке по умолчанию (в данном случае — на английском языке). Кроме того, можно обновить имя и описание сайта после создания сайта. Затем вы создадите узлы навигации и содержимое нижнего колонтитула на английском языке.

После настройки сайта на английском языке пользователь с испанским языком изменяется вручную и преобразует название, описание, навигацию и содержимое нижнего колонтитула на испанском языке. Пользователь с немецким языком в качестве предпочтительного для немецкого языка. После перевода контента он будет отображаться для всех пользователей этих предпочитаемых языков. 

> [! ОТМЕЧАЕТ
>- Пользователи, выполняющие перевод контента сайта для предпочитаемых языков, должны быть членами группы владельцев сайта или иметь эквивалентные разрешения для сайта.
>- При изменении имени сайта, навигации или нижнего колонтитула на языке по умолчанию соответствующий переведенный элемент на другом языке не обновляется автоматически, если не выбрано перезапись существующих переводов сайта. В этом случае переведенный элемент заменяется обновлением на языке по умолчанию и потребуется вручную выполнить его перевод. Чтобы перезаписать переводы, перейдите на страницу "Языки сайта" для параметра язык по умолчанию и выберите пункт Показать дополнительные параметры. Затем на слайде переключитесь в режим перезаписи для перевода. Этот параметр не применяется к содержимому страниц или новостей.

### <a name="to-view-the-fully-translated-site-in-a-specific-language"></a>Просмотр полностью переведенного сайта на определенном языке

Дополнительные сведения об установке языковых и региональных параметров приведены в статье [изменение личных языков и региональных параметров](https://support.microsoft.com/en-us/office/change-your-personal-language-and-region-settings-caa1fccc-bcdb-42f3-9e5b-45957647ffd7). Для просмотра переведенных страниц рекомендуется использовать отдельную учетную запись или другой пользователь с разными языковыми параметрами.
